FTP와 SFTP는 서로 다른 두 파일 전송 프로토콜이며 이 둘의 주요 차이점은 파일 전송과 관련된 보안입니다.

-FTP (File Transfer Protocol)는 인터넷을 통해 파일을 교환하기 위해 일반적으로 사용되는 프로토콜입니다.
FTP는 인터넷의 TCP/IP 프로토콜을 사용하여 데이터 전송을 가능하게 합니다.
FTP는 종종 SSL/TLS로 보안되는 클라이언트-서버 아키텍처를 사용합니다.
FTP는 안정적이고 효율적인 데이터 전송으로 원격 컴퓨터를 통한 파일 공유를 촉진합니다.

인터넷과 같은 TCP/IP 네트워크는 FTP 또는 파일 전송 프로토콜을 사용하여 한 컴퓨터에서 다른 컴퓨터로 파일을 전송합니다.
FTP는 서버 및 클라이언트 기반 아키텍처에서 작동합니다.
즉, 클라이언트는 주어진 시간에 서버의 모든 정보에 액세스할 수 있습니다.
일부 서버는 비밀번호로 보호되므로 ID와 비밀번호를 입력한 후에만 정보에 액세스할 수 있습니다.

인터넷에서 mp3 음악 또는 무료 소프트웨어를 다운로드하는 것은 FTP의 두가지 일반적인 예입니다.
클라이언트는 "정보"에 액세스하고 다운로드하기 위해 ID나 비밀번호가 필요하지 않습니다.
반면에 "이메일"에 액세스하는 것은 SFTP의 예의며 "서버의 정보에 액세스"하려면 ID와 비밀번호가 필요합니다.

FTP는 데이터와 제어를 위해 두 개의 별도 채널을 사용하며 둘 다 암호화되지 않습니다.
이는 채널 중 하나의 정보를 가로 채서 액세스할 수 있음을 의미합니다.
암호화되지 않은 채널을 사용하는 것은 엄청난 보안 허점이며 FTPS 및 SFTP 와 같은, 보다 안전한 통신 모드를 설계했습니다.

-SFTP
SFTP (SSH 파일 전송 프로토콜)는 SSH 프로토콜 제품군의 일부입니다.
보안 파일 전송 프로토콜이라고도 하며 SSH를 통한 보안 파일 전송을 구현합니다.
SSH 키를 포함한 SSH 프로토콜의 전체 보안 및 인증 기능을 지원합니다.

SSH는 "원격 서버의 모든 셸 계정에 액세스 할 수 있는" 안전한 방법입니다.
SFTP를 사용하여 통신된 정보는 먼저 "작은 패킷으로 분할"되며 FTP와 달리 SFTP는 "데이터 및 제어를 위해 하나의 채널만 사용"합니다.
두 컴퓨터간에 정보를 공유하기 전에 SFTP는 클라이언트의 신원을 확인하고 일단 보안 연결이 설정되면 암호화된 정보를 전송합니다. (암호화 암호가 사전 정의 됨)

FTP와 SFTP는 둘 사이에 많은 차이가 있습니다.

FTP는 파일 전송 프로토콜을 의미하고 SFTP는 보안 파일 전송 프로토콜을 의미합니다.
FTP는 포트 21을 사용하지만 SFTP는 포트 22를 사용합니다.
SFTP는 파일을 완전한 보안으로 공유하는 반면 FTP는 보안없이 데이터를 업로드하거나 다운로드합니다.
Filezilla는 FTP에 적합하고 winSCP는 SFTP에 가장 적합합니다.
SFTP는 약간의 프라이버시를 제공하지만 FTP는 일반 텍스트로 데이터를 보냅니다.
FTP는 다른 사람이 사용하지만 SFTP는 공유 호스팅의 경우 22 포트가 열려 있지 않기 때문에 서버 소유자가 사용합니다.
SFTP는 완전히 안전하고 만족스러운 반면 FTP는 덜 안전합니다.
